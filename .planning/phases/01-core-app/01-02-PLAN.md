<task type="auto">
  <name>Task 1: Create GitHub Actions release workflow with signing</name>
  <files>.github/workflows/release.yml, src-tauri/tauri.conf.json</files>
  <action>
  1. Create `.github/workflows/release.yml` with a matrix build strategy:

  **Trigger:** `on: push: tags: ['v*']`

  **Jobs:** Single `build-and-release` job with matrix:
  ```yaml
  strategy:
    fail-fast: false
    matrix:
      include:
        - platform: macos-latest
          args: '--target aarch64-apple-darwin'
        - platform: macos-latest
          args: '--target x86_64-apple-darwin'
        - platform: windows-latest
          args: ''
  ```

  **Steps for each matrix entry:**
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: '22'`
  3. Install pnpm: `corepack enable && corepack prepare pnpm@latest --activate` (or use `pnpm/action-setup@v4`)
  4. `dtolnay/rust-toolchain@stable` — for macOS aarch64 target, also add `targets: aarch64-apple-darwin` or `targets: x86_64-apple-darwin` as appropriate
  5. `swatinem/rust-cache@v2` for Cargo caching
  6. `pnpm install` for frontend deps
  7. `tauri-apps/tauri-action@v0` with:
     - `tagName: 'v__VERSION__'`
     - `releaseName: 'CUID Generator v__VERSION__'`
     - `releaseDraft: true` (so user can review before publishing)
     - `prerelease: false`
     - `args: ${{ matrix.args }}`

  **Environment variables for signing (all from GitHub Secrets):**
  - macOS: `APPLE_CERTIFICATE`, `APPLE_CERTIFICATE_PASSWORD`, `APPLE_SIGNING_IDENTITY`, `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID`
  - Windows: `AZURE_CLIENT_ID`, `AZURE_TENANT_ID`, `AZURE_CLIENT_SECRET`
  - Both: `GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}`

  Conditionally set signing env vars: macOS env vars only on `runner.os == 'macOS'`, Windows env vars only on `runner.os == 'Windows'`. Or simply set all — unused vars are harmless.

  2. Update `src-tauri/tauri.conf.json` to add Windows signing command:
  ```json
  {
    "bundle": {
      "windows": {
        "signCommand": "relic sign --config relic.yml -k azure %1"
      }
    }
  }
  ```

  NOTE: The `signCommand` is only invoked during `tauri build` on Windows when the `relic` tool and Azure Key Vault are configured. It will be silently skipped in local dev builds where `relic` is not installed. If the user has not set up Windows signing yet, this is safe to include — it only activates in CI with the proper env vars.

  3. Ensure the workflow also runs `pnpm test` as a pre-build step to catch regressions before building artifacts.

  IMPORTANT: Use `tauri-apps/tauri-action@v0` (not v1 or latest) — this is the official action for Tauri v2 as confirmed in research.
  </action>
